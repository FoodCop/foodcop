The biggest mistake would be trying to use **one prompt** for all Masterbots.

What you actually want is:

## Layer 1 — Global Brand DNA

(shared by every image)

## Layer 2 — Profile DNA

(Jun, Aurelia, Sebastian, etc.)

## Layer 3 — Dish DNA

(the actual food)

This gives you consistency while still making each Masterbot visually distinct.

# API TEMPLATE

Generate the final prompt by assembling:

```text
[GLOBAL STYLE]

[PROFILE STYLE]

[PROFILE LIGHTING]

[DISH]

Professional commercial food photography.
```

Example:

```text
[GLOBAL STYLE]

[AURELIA STYLE]

[AURELIA LIGHTING]

A gourmet loaded hot dog featuring artisan sausage, caramelized onions, specialty sauces and crispy toppings.

Professional commercial food photography.
```

This structure scales extremely well because you can store:

```json
{
  "masterbot": "Aurelia Voss",
  "profile": "Street Explorer",
  "style": "...",
  "lighting": "...",
  "dishes": [...]
}
```

Then generate **10 dishes × 7 Masterbots = 70 unique images** with a consistent visual identity and very little prompt engineering per image.
